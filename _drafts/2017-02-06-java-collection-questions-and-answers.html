A: For Sun Java 6 src: int newCapacity = (oldCapacity * 3)/2 + 1;


Q: Why HashMap load factor is 0.75
A: Offers a good tradeoff between time and space costs. Higher values decrease the space overhead but increase the lookup cost

Q: How HashMap bucket table size is calculated?
A: In power of 2

Q: How HashMap bucket index calculated?
A: In short: hash & (bucket table length-1)
Long story: Because length is always a power of 2, length - 1 will be a bitmask use to AND with hash, which results the index.
